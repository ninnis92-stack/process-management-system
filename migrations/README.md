This folder contains Alembic migration scripts for the application.

Current state:
- An initial empty revision `0001_initial.py` is provided. It does not
  execute DDL; it exists so you can stamp the DB as up-to-date if the
  database schema already matches the models.

Recommended usage:

1. Install Flask-Migrate/Alembic in your environment:

```bash
pip install Flask-Migrate alembic
```

2. Initialize migrations (if you haven't already) and generate an autogen
   migration based on current models:

```bash
export FLASK_APP=run.py
flask db init     # only if `migrations/` doesn't exist yet
flask db migrate -m "autogenerate initial"
```

3. If your target DB already contains the current schema (e.g., production),
   mark it without applying DDL:

```bash
flask db stamp head
```

4. Otherwise, apply the migration:

```bash
flask db upgrade
```

Notes:
- Autogenerated migrations often require manual tweaks for complex
  operations or data migrations. Review diffs before applying in
  production.
- This repo includes a minimal `0001_initial.py` that is intentionally
  empty to avoid unintentionally altering existing DBs; prefer generating
  a proper migration in your environment using `flask db migrate`.
