# Process Management Prototype

A lightweight Flask application for tracking and managing cross‑department
requests. It demonstrates a small but feature‑rich workflow engine with
notifications, themes, SSO support, administrative controls, and extensible
integration endpoints.

---

## Features

- **Request workflows** with configurable statuses and transition rules.
- **Notification system** supporting in‑app and email delivery.
- **Admin UI** for managing users, workflows, status options, and feature flags.
  Toggle checkboxes display action‑oriented labels so you always know what the
  click will do.
- **Per‑user themes & "vibes"** including dark mode, with server‑side persistence
  and a global no‑vibe mode for consistency.
- **SSO/OIDC integration** with optional admin role syncing.
- **External integrations** via a simple provider architecture plus webhook
  subscriptions (`request.status_changed` event is emitted on transitions).
